`UnityAcousticSimulator` — это набор компонентов для Unity, предназначенный для приблизительного моделирования акустики помещения, поведения акустических систем, микрофонов, материалов, сигнального тракта и позднего поля прямо внутри сцены.

Проект позволяет собрать виртуальную модель зала и получить практическую оценку:
- распределения звукового давления в помещении;
- влияния направленности колонок и микрофонов;
- влияния материалов поверхностей;
- риска обратной связи;
- влияния микшера и сигнальных линий;
- проблемных зон в зале.

Система ориентирована на быструю инженерную/сценическую оценку в Unity, а не на сертифицированный акустический расчёт.

## Что входит в систему

В репозитории есть следующие основные компоненты:

- `AcousticSimulator.cs` — центральный симулятор
- `AcousticSpeaker.cs` — описание акустической системы
- `AcousticMicrophone.cs` — описание микрофона
- `AcousticMaterial.cs` — описание акустического материала/поверхности
- `AcousticSignalWire.cs` — описание сигнальной линии
- `AcousticListenerProbe.cs` — контрольная точка прослушивания
- `Onyx24Mixer.cs` — модель микшера
- `Onyx24MixerPreset.cs` — пресет микшера (`ScriptableObject`)
- `BandLimitedRoomOperatorSolver.cs` — расчёт позднего поля / diffuse late field
- `AcousticsCommon.cs` — общие типы, полосы, утилиты и математика
- `Editor/AcousticSimulatorEditor.cs` — кастомный инспектор симулятора
- `Editor/Onyx24MixerEditor.cs` — кастомный инспектор микшера

1. Акустические системы (AcousticSpeaker)

Компонент описывает источник звука и его направленность.

полярность;

электрическая задержка;

Это позволяет моделировать:

порталы;

сценические мониторы;

сателлиты;

распределённые системы.

2. Микрофоны (AcousticMicrophone)

Используются полярные диаграммы наподобие:

Это даёт возможность оценивать:

Таким образом можно задавать не только отражение, но и:

4. Сигнальные линии (AcousticSignalWire)

Компонент нужен для моделирования тракта.

Параметры включают:

Это полезно для цепочек:

5. Точки прослушивания (AcousticListenerProbe)

Такие точки удобно ставить:

7. Позднее поле (BandLimitedRoomOperatorSolver)

Дополнительный компонент для расчёта позднего диффузного поля помещения.

Настраиваются:

влияние стен;

Этот модуль нужен, когда обычного прямого звука и ранних отражений недостаточно и нужно более “зальное” поведение.

8. Центральный симулятор (AcousticSimulator)

Это основной управляющий компонент системы.

Он:

строит границы помещения;

считает прямой звук;

считает отражения;

учитывает материалы;

учитывает transmission по сегменту;

считает структурную передачу;

при наличии late solver строит позднее поле;

анализирует микрофоны;

анализирует listener probes;

анализирует сетку помещения;

ищет hotspots;

пишет отчёт в консоль;

строит debug rays и gizmos.

У него есть context menu команды:

Auto Collect Scene Objects

Simulate Acoustics

Clear Results

Как установить
Вариант 1. Прямое копирование в проект

Скачайте репозиторий.

Поместите его в папку проекта Unity, например:
Assets/UnityAcousticSimulator/

Дождитесь компиляции скриптов.

Убедитесь, что ошибок компиляции нет.

Вариант 2. Как внешний модуль

Можно хранить репозиторий отдельно и синхронизировать его вручную или через git submodule.

Требования к сцене

Чтобы система считала адекватно, желательно:

чтобы у объектов помещения были Collider;

чтобы стены, пол, потолок и крупные конструкции были представлены в сцене;

чтобы объекты, участвующие в акустике, попадали в acousticLayerMask;

чтобы колонки и микрофоны были корректно ориентированы по transform.

Без коллайдеров и корректной геометрии расчёт будет сильно упрощённым.

привяжите bus outputs к AcousticSpeaker;

Рекомендуемый рабочий порядок

Полезные советы
Следите за ориентацией

Одна из самых частых причин неправильного результата — неверный transform.forward или неверная выбранная ось излучения/приёма.

Не делайте зал слишком пустым

Если в реальности есть:

сцена,

кулисы,

экраны,

мебель,

подвес,

массивы акустики,

пультовая зона,

то их лучше отразить в геометрии сцены.

Начинайте с умеренных настроек

Для первых прогонов не стоит сразу сильно увеличивать:

raysPerSpeaker

maxReflectionOrder

качество late solver

Сначала лучше получить быстрый черновой расчёт, потом повышать детализацию.

Ставьте достаточно listener probes

Пара точек почти всегда недостаточна.
Лучше иметь сетку важных позиций в зале.

Используйте presets для микшера

Если сравниваете несколько конфигураций канального тракта, Onyx24MixerPreset сильно упрощает работу.

Ограничения

Важно понимать границы применимости:

это не ГОСТ/ISO-сертифицированный акустический пакет;

расчёт основан на приближениях;

точность сильно зависит от геометрии и коллайдеров;

материалы заданы упрощёнными полосовыми коэффициентами;

результат нужно сверять с реальными измерениями и практикой на площадке.

То есть система хороша для:

предварительной оценки;

поиска проблемных мест;

виртуальной отладки конфигурации;

обучения и экспериментов.

Но она не заменяет полевые измерения, real-time tuning и инженерный Smaart/ARTA/измерительный цикл.

преподавателям;

Пример типового применения

Пример реального сценария:

Собрать приблизительную модель школьного/актового/концертного зала.

Итог

UnityAcousticSimulator — это практический набор Unity-компонентов для моделирования сцены и зала, где учитываются:

источники звука;

микрофоны;

акустические материалы;

сигнальные линии;

микшер;

позднее поле;

listener probes;

hotspots и риск обратной связи.

Проект подходит для быстрого прототипирования, инженерной прикидки, визуального анализа и обучения внутри Unity.

Если использовать его итеративно и осмысленно, он позволяет заранее находить слабые места конфигурации и приходить на площадку с более подготовленным звуковым решением.
